# 四、重要事件及收获

- 规范问题。部分成员不愿意了解规范，时间紧任务重，无法对所有的PR进行Code Review，或是对项目进行重构，代码只做到了基本的层次和结构清晰，但是称不上优雅，有时间需要进行重构。
- 分工问题。初次现场团队合作编程，分工是个大问题。各个成员对技术的掌握能力不同，故在分配时合理分到自己能解决的任务，以及如何拆分总任务，对团队协作是一个巨大的挑战。
- 技术问题。
    - 核心逻辑和算法强相关，在压力和讨论之下，我们仍然选择拒绝参考或借鉴他人的代码，选择自己从零开始构思算法结构与具体实现，虽然过程可能比较辛苦，并且没有把握超过现成的开源代码，但是开发人员在此过程中都有所收获。无论是处理gbk三个字节的编码，或是对输入输出进行正确性校验，都是全新的知识与体会。
    - 平时做项目都是自己安排时间，ddl一般情况下比较松，这么紧张的时间下对各成员掌握的技术提出了更高的要求，平时学习到技术的一些不太清楚的地方也都暴露出来。~~摸鱼之人无所遁形！~~
    - 我们的项目部署在**服务器**上，部署过程中，由于开发测试时使用*Axios*访问的是*http*的网址，而部署服务器上有*https*的证书，部署完成后会出现CORS的问题，在手机上的默认浏览器(华为、小米系统内置的浏览器)中，由于安全要求低，不会出现CORS的问题，但是在edge这类对安全性有严苛要求的浏览器上，会出现CORS的问题，简单讨论过后，我们最终使用nginx反向代理解决这一问题。另外，后台部署时采用Docker进行容器化部署，但是Docker官方的GO 1.19镜像访问速度不尽人意，最后多次尝试后才成功构建镜像进行部署。
- 配合问题。成员间之前没有多人现场编程的经历，现场编程时CPU过载，难以兼顾到每位成员，时间也一度来不及。
- 收获：日后需要多加强沟通，学习时也要更加认真。**拒绝摸鱼！**(学会怎么把代码写的更优雅，而不能跑起来就行！)
